# Bank Account Withdrawal Code Improvement Exercise

This repository contains my submission for the Bank Account Withdrawal Code Improvement Exercise.

The objective of my submission was to preserve the existing business functionality while improving the overall quality of the implementation.

Some of the areas I focused on include:

- Separation of concerns
- Moving business logic out of the controller
- Reducing coupling through interfaces
- Improving the service contract
- Replacing magic strings with enums and constants
- Returning more meaningful HTTP responses
- Addressing the concurrency issue in the withdrawal logic

A separate PDF accompanies this submission, outlining my approach, implementation choices, and additional improvements I would consider in a production environment.

Thank you for taking the time to review my submission.
